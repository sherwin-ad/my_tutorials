## 1. Basic Concepts and Values of Cloud Computing

### Cloud Deployment Models

**Private cloud**

- Owned and managed for exclusive use by a single organization

**Public cloud**

- Owned and managed by a third party cloud service provider and shared with multiple
  organizations over the Internet

**Hybrid cloud**

- A combination of public and private clouds, viewed as a single cloud externally



### Cloud Service Models

**Infrastructure as a service (IaaS)**

- Intended for: enterprises (or a few individual users)
- Offerings: infrastructure services such as computers, storage devices, and networks
- Examples: AWS, Google Cloud, Azure, and OpenStack

**Platform as a service (PaaS)**

- Intended for: developers
- Offerings: database middleware, MySQL, MongoDB, and Java
- Examples: Cloud Foundry and OpenShift

**Software as a service (SaaS)**

- Intended for: enterprise/individual users
- Examples: enterprise application services, email, instant messaging (IM), and microblogs

#### üîπ Highlights:

- Fully managed, internet-accessible
- Unlimited storage (up to 48 TB per object)
- 99.9999999999% reliability
- Event triggering and cost-effective

#### üîπ Key Concepts:

#### üîπ Features:

- **Permission Control**: ACLs and bucket policies

#### üîπ Billing:

- Free inbound traffic and intra-region transfers

### 3. **Elastic Volume Service (EVS)**

#### üîπ Highlights:

- Block storage for ECS
- 3-copy redundancy in one AZ
- Up to 32 TB per disk
- Backup stored in OBS

#### üîπ Reliability Features:

- **Three-copy redundancy**: Ensures data durability
- **Backups**: Restore or create disks from backups
- **Snapshots**: Point-in-time recovery

#### üîπ Lifecycle:

- Create ‚Üí Attach ‚Üí Use ‚Üí Snapshot ‚Üí Detach ‚Üí Delete

#### üîπ Use Cases:

- System disks, databases, durable services
- Sensitive apps with encryption

#### üîπ Billing:

- Yearly/monthly or pay-per-use
- Charged based on allocated size
- Region-based pricing

------

### 4. **Local Disks**

#### üîπ Highlights:

- Ultra-low latency and high IOPS
- Available only for specific ECS types (e.g., i3, d6)
- No extra fees post ECS purchase
- Data loss risk if ECS stops or crashes

------

### 5. **Scalable File Service (SFS)**

#### üîπ Highlights:

- Supports NFS and partial CIFS
- 3-copy redundancy, 99.99999999% durability
- Elastic and linear performance scaling

#### üîπ Network Management:

- ECS must be in the same VPC as SFS
- Supports internal access and Direct Connect

It explains how to **build secure, scalable, and connected cloud networks**, enabling communication between cloud resources and external systems.

------

## üìö **Detailed Breakdown**

### 1. **Virtual Private Cloud (VPC)**

#### üîπ Overview:

- A **logically isolated network** for cloud resources (ECS, RDS, etc.)
- Acts like a **LAN on the cloud**
- Custom IP ranges, subnets, and routing

#### üîπ Components:

- **CIDR block**: Defines IP range
- **Subnets**: Enable refined network control
- **Route tables**: Manage traffic flow between subnets

#### üîπ Use Case:

- Isolate environments (e.g., production vs. test systems)

------

### 2. **Access Control**

#### üîπ Security Groups:

- Protect **instances**
- Define **inbound/outbound rules**
- Rules apply to all instances in the group

#### üîπ Network ACLs:

- Protect **subnets**
- Deny all traffic by default until rules are added
- One ACL per subnet; ACLs can span multiple subnets

------

### 3. **Cloud Network Connectivity**

#### üîπ VPC Peering:

- Connects **two VPCs** in the same region
- Requires **route configuration**
- Enables **private communication** between VPCs

#### üîπ VPN & Direct Connect:

- For **cross-region** or **on-premises** connectivity

------

### 4. **Elastic IP (EIP)**

#### üîπ Overview:

- Public IP for Internet access
- Can be **bound/unbound** to cloud resources
- Supports **bandwidth and traffic billing**
- Flexible bandwidth adjustment

#### üîπ Use Case:

- Assign EIP to ECS for Internet access

------

### 5. **NAT Gateway**

#### üîπ Overview:

- Enables **multiple ECSs** to share one EIP
- Supports **SNAT** and **DNAT**

#### üîπ SNAT (Source NAT):

- ECSs access the Internet using shared EIP

#### üîπ DNAT (Destination NAT):

- Internet users access ECS services via shared EIP

#### üîπ Configuration:

- Create NAT gateway
- Define SNAT/DNAT rules
- Set default route to NAT gateway

------

### 6. **Best Practices**

#### üîπ Architecture Example:

- VPC with subnets for frontend, application, and database
- NAT Gateway for Internet access
- VPN for on-premises connectivity

------

### 7. **Quizzes & Exercises**

Includes practical tasks and questions to reinforce:

- VPC creation
- Security group setup
- VPC peering
- EIP and NAT gateway configuration

------

### 8. **Summary**

- VPCs isolate and organize cloud networks
- Security groups and ACLs secure resources
- VPC peering and VPN connect networks
- EIP and NAT Gateway enable Internet access

## üìö **Detailed Breakdown**

### 1. **Why Scalability Matters**

- Traffic fluctuates due to business cycles, promotions, or unpredictable demand.
- Scalable systems:
  - Improve reliability and performance
  - Optimize resource usage
  - Reduce costs

------

### 2. **Scalability Methods**

#### üîπ **Scaling Up**

- Increase resources (CPU, memory) on a single server.
- Simple but limited and prone to bottlenecks.

#### üîπ **Scaling Out**

- Add more servers to distribute load.
- Ideal for **stateless applications**.
- Requires coordination and distributed design.

------

### 3. **Distributed Deployment Models**

- **Task polling**
- **Synchronous coordination**
- **Request distribution**

#### üîπ Benefits:

- Fault recovery
- Load balancing
- Resource pooling

------

### 4. **Elastic Load Balance (ELB)**

#### üîπ Features:

- Distributes traffic across backend servers
- Supports Layer 4 and Layer 7 load balancing
- Built-in health checks
- Works with Auto Scaling

#### üîπ Components:

- **Listener**: Defines protocol and port
- **Backend Server Group**: Hosts ECS instances
- **Forwarding Rules**: URL, domain, headers, etc.

#### üîπ Algorithms:

- Weighted round robin
- Weighted least connections
- Source IP hash

------

### 5. **Health Checks**

- Prevent routing to unhealthy servers
- TCP or HTTP-based checks
- Use dedicated health check pages

------

### 6. **Cloud Eye**

#### üîπ Features:

- Real-time monitoring of cloud resources
- Configurable alarms
- Visual dashboards
- API integration for third-party tools

------

### 7. **Auto Scaling**

#### üîπ Features:

- Automatically adds/removes ECS instances
- Supports dynamic and scheduled scaling
- Replaces unhealthy instances
- Works with ELB for traffic distribution

#### üîπ Scaling Policies:

- **Dynamic**: Based on performance metrics
- **Scheduled**: Based on predictable workload patterns

#### üîπ Configuration Steps:

1. Create a scaling template
2. Create a scaling group
